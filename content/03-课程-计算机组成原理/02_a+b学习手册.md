## 序章：我们的主角是谁？

```c
#include <stdio.h>
int main() {
    int a, b;
    scanf("%d %d", &a, &b);   // 第①步：从键盘读入两个整数
    printf("%d", a + b);      // 第②步：算出 a+b，打印到屏幕
    return 0;                 // 第③步：结束，把控制权还给操作系统
}
```

别看它只有几行，它要正常跑起来，**整台计算机几乎所有部件都得出力**。我们先剧透一下它这一生要闯哪些"关"：

```
   你写的 a+b.c
        │  ① 编译/汇编（第1章：翻译成机器能懂的二进制）
        ▼
   a+b 可执行程序（一串01指令 + 数据）
        │  ② 装入内存（第3章：程序住进存储器）
        ▼
 ┌──────────────────────────────────────────────────────┐
 │  CPU 开始按地址自动执行（第5章：取指→译码→执行）      │
 │   ├─ 需要"加法怎么算"           → 第2章 运算器          │
 │   ├─ 需要"a、b、指令放哪、咋取快" → 第3章 存储器/Cache   │
 │   ├─ 需要"用哪条指令、怎么找操作数"→ 第4章 指令系统      │
 │   ├─ 数据在部件间跑，走哪条路     → 第6章 总线           │
 │   ├─ scanf 从键盘读、printf 往屏幕写 → 第7章 外设        │
 │   └─ 慢键盘怎么配合快CPU          → 第8章 I/O方式        │
 └──────────────────────────────────────────────────────┘
        │  ③ 如果要同时算一万个 a+b 更快？ → 第9章 并行/多核
        ▼
   屏幕显示出 a+b 的结果
```

🎓 **专业版**：一个高级语言程序的执行，本质是"源程序经翻译变为机器指令序列，存入主存，由 CPU 在时序控制下按地址自动取指并执行，期间通过总线在运算器、存储器、I/O 之间传送地址/数据/控制信息"。

🌱 **大白话**：你写的代码就像一张菜谱，得先翻译成厨师（CPU）认识的暗号，存进冰箱（内存），厨师再照着暗号一步步把菜（a+b 的结果）做出来端给你看。

## 🏁 全书合龙：a+b 的"一生"完整回放

```
你写下 a+b.c                                         【第1章: 高级语言级, 冯诺依曼机器】
   │ gcc 编译 / 汇编器汇编                            【第4章: 翻译成机器指令 LOAD/ADD/STORE】
   ▼
机器指令(补码表示的数 + 01指令)                       【第2章: a,b存成补码】
   │ 装入主存, 必要时虚存映射                          【第3章: 住进内存, Cache加速, 虚存扩容】
   ▼
┌──────────────── CPU 循环 ────────────────┐
│ 取指: 按PC取指令入IR, PC+1                 │         【第5章: 指令周期-取指】
│ 译码: 认出这是ADD                         │
│ 执行: ALU用补码加法算a+b, 防溢出           │         【第2章+第5章: 运算器执行】
│   ↑ 取a/b、回写, 经总线传输, 仲裁定时       │         【第6章: 总线】
└───────────────────────────────────────────┘
   │ scanf 输入(键盘) / printf 输出(屏幕)             【第7章: 外设硬件】
   │   慢外设用 中断/DMA 与CPU高效配合                 【第8章: I/O方式】
   ▼
屏幕显示 a+b 的结果 ✅
   │ 想同时算一万个? → 多核/流水/超线程                【第9章: 并行】
   ▼
现代真实机器(酷睿/安腾)把这一切综合实现               【第10章: 实例】
```

🎓 **一句话总纲**：计算机组成原理 = "存储程序 + 程序控制"思想下，五大部件如何用合适的数据表示、运算方法、存储层次、指令系统、控制时序、总线互连和 I/O 机制，协同把内存里的指令自动执行完。

🌱 **一句话大白话**：你写的每一行代码，最后都变成内存里一堆开关(0/1)，CPU 像个超守纪律的工头，照着这些开关一拍一拍把活干完，再把结果给你看——这门课就是把这台"自动工厂"每个车间拆开讲清楚。
