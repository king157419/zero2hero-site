# 费曼学习法教学Skill

## 触发方式
手动调用：`/feynman [主题]`

## 核心使命
将复杂概念转化为12岁小孩也能懂的教学内容，生成完整的、结构清晰的、问题导向的教学文档。

---

## 七大教学原则

### 原则1：问题导向（核心原则）
每个概念/技术的讲解必须包含：
1. **背景**：当时存在什么问题？
2. **动机**：为什么需要这个技术？
3. **解决方案**：这个技术如何解决问题？
4. **突破**：相比之前的方法，突破了什么瓶颈？
5. **局限**：这个技术还有什么局限？

**示例**：讲解Self-Attention
- 问题：RNN处理长序列时的记忆衰减问题
- 动机：需要让模型"同时看到"所有位置的信息
- 解决方案：通过Q、K、V机制计算任意两个位置的关联
- 突破：打破了顺序处理的限制，可以并行计算
- 局限：计算复杂度是O(n²)

### 原则2：结构组织（金字塔模型）
```
顶层：核心概念（1-2句话）
第二层：基础知识（3-5个小节）
第三层：深入理解（对比、应用）
第四层：高级扩展（可选）
```

### 原则3：渐进性（三段式）
```
第一段：基础概念（简单、直观）
第二段：核心原理（对比、深入）
第三段：实际应用（例子、案例）
```

### 原则4：类比质量（三层验证）
```
准确性：类比不能有逻辑漏洞
可理解性：贴近日常生活经验
局限性：明确说明类比的边界
```

### 原则5：深度控制（80/20法则）
```
80%篇幅：核心概念和原理
20%篇幅：细节和扩展内容
```

### 原则6：互动验证（分级设计）
```
基础题（30%）：检验概念理解
中等题（50%）：检验原理理解
进阶题（20%）：检验应用理解
```

### 原则7：Obsidian格式美观
```
- 使用frontmatter（title, date, tags, status等）
- 使用callout（note, tip, example, warning, question等）
- 使用wikilink（[[链接]]）
- 使用表格、代码块、数学公式
- 使用折叠标题、高亮、引用
- 确保在Obsidian阅读模式下美观
```

## 常见问题处理

### Q1: 如果概念太复杂怎么办？
**A**: 拆分成更小的子概念
- 每个子概念单独用12岁版解释
- 每个子概念都说明解决了什么问题
- 最后再组合起来

### Q2: 如果找不到好的类比怎么办？
**A**: 使用多个部分类比
- 明确说明每个类比的适用范围
- 提供对比表格
- 强调类比的局限

### Q3: 如果用户要求更深入怎么办？
**A**: 在文档末尾添加"深入阅读"部分
- 提供进阶资源链接
- 但主体内容仍保持易懂
- 遵循80/20法则

### Q4: 如何确保问题导向？
**A**: 每个概念都要回答5个问题
- 背景：当时存在什么问题？
- 动机：为什么需要这个技术？
- 解决方案：这个技术如何解决问题？
- 突破：相比之前的方法，突破了什么瓶颈？
- 局限：这个技术还有什么局限？

### Q5: 如何控制文档长度？
**A**: 根据主题复杂度自然展开
- 不要过度深入细节（80/20法则）
- 但要确保完整性（所有必要部分都有）
- 如果内容太多，考虑拆分成多个文档

## 注意事项

1. **不要过度简化**：12岁版解释是为了建立直觉，但后续要引入准确术语
2. **不要跳过问题导向**：这是核心原则，每个概念都要说明解决了什么问题
3. **不要忽略局限**：每个技术都有局限，要诚实说明
4. **不要忘记分级**：思考题必须分级，答案必须完整
5. **不要忽视格式**：Obsidian格式很重要，确保美观
6. **分块写入文档**：生成1500-2000行的长文档时，必须分块写入
   - 第一次用Write创建文件，只写前50-100行（开篇+目录+第一部分开头）
   - 后续用Edit逐步追加内容，每次追加50行左右
   - 这样可以避免内容过长导致的写入失败
   - 遵循chunked write protocol
